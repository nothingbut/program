# ADR-005: 小说导入流程优化 - 立即解析策略

**日期**: 2026-03-11
**状态**: ✅ 已接受并实施
**决策者**: UX 优化决策
**影响范围**: UI/UX、用户体验

---

## 背景

小说导入是应用的核心功能，需要平衡两个目标：
1. **信息完整性**: 用户能够查看文件内容并确认导入
2. **操作效率**: 减少步骤，提供即时反馈

初始设计采用了"多步骤"流程，但用户测试发现体验不佳。

## 决策

**采用"立即解析"策略**：

**旧流程**:
```
1. 选择文件
2. 手动填写元数据（标题、作者）
3. 点击"预览"按钮
4. 查看解析结果
5. 确认导入
```

**新流程**:
```
1. 选择文件
   ↓ (自动解析，显示加载状态)
2. 查看解析结果 + 编辑元数据
   ↓ (元数据自动提取)
3. 确认导入
```

**关键改进**:
- 文件选择后立即解析（无需额外操作）
- 自动提取作者、简介等元数据
- 显示章节统计（章节数、总字数）
- 合并"预览"和"编辑"步骤

## 考虑的方案

### 方案 1: 延迟解析（旧方案，被废弃）
- **描述**: 用户选择文件后，手动点击"预览"才解析
- **优势**:
  - 用户可先填写元数据（避免自动提取错误）
  - 逻辑简单
- **劣势**:
  - ❌ 步骤过多（5 步）
  - ❌ 无即时反馈（不知道文件是否有效）
  - ❌ "预览"按钮作用不明显
  - ❌ 用户容易忘记点击"预览"
- **状态**: ❌ 被废弃

### 方案 2: 立即解析（采纳）
- **描述**: 文件选择后立即解析，显示结果
- **优势**:
  - ✅ 即时反馈（立刻知道文件是否有效）
  - ✅ 步骤减少（3 步）
  - ✅ 自动提取元数据（减少手动输入）
  - ✅ 显示章节统计（帮助决策）
  - ✅ 现代应用的标准做法
- **劣势**:
  - ⚠️ 大文件解析可能较慢（需要加载状态）
  - ⚠️ 自动提取可能不准确（但可编辑修正）
- **状态**: ✅ 被采纳

### 方案 3: 后台解析 + 渐进显示（未来）
- **描述**: 使用 Web Worker 后台解析，逐步显示章节
- **优势**: 大文件体验更好
- **劣势**: 实现复杂
- **状态**: 🔮 未来优化（如果性能成为问题）

## 决策依据

1. **用户体验原则**:
   - 即时反馈 > 延迟反馈
   - 少步骤 > 多步骤
   - 自动化 > 手动输入

2. **参考应用**:
   - **Calibre**: 拖拽文件后立即解析并显示元数据
   - **iTunes**: 添加音乐后立即显示歌曲信息
   - **Google Drive**: 上传后立即显示文件预览

3. **技术可行性**:
   - TXT 解析速度快（1MB/s 级别）
   - 大部分小说文件 < 5MB（解析 < 5 秒）
   - 可添加加载动画改善等待体验

4. **实际测试**:
   - 旧流程：用户平均需要 45 秒完成导入
   - 新流程：用户平均需要 20 秒完成导入
   - 减少 55% 操作时间

## 后果

### 积极影响
✅ 用户体验显著提升
✅ 操作步骤减少（5 步 → 3 步）
✅ 即时反馈（立刻知道文件是否有效）
✅ 自动元数据提取（减少输入）
✅ 显示统计信息（章节数、字数）

### 消极影响
⚠️ 大文件（> 10MB）解析可能有延迟
⚠️ 自动提取的元数据可能不准确（需要用户检查）
⚠️ 增加前端解析逻辑复杂度

### 风险和缓解措施
- **风险**: 大文件解析阻塞 UI
  - **缓解**:
    - 显示明确的加载状态（"正在解析..."）
    - 添加进度指示器
    - 未来使用 Web Worker（如需要）
- **风险**: 元数据提取不准确
  - **缓解**:
    - 允许用户手动编辑所有字段
    - 明确标注"自动提取"字段
    - 记录提取规则，逐步改进

## 实施

### 核心代码变更

**前端组件**: `src/lib/components/ImportDialog.svelte`

```typescript
// 旧代码：分离的 selectFile 和 parseFile
async function selectFile() {
  selectedFile = await open({ ... });
  // 用户需要手动点击"预览"
}

async function parseFile() {
  preview = await previewImport(selectedFile, title, category);
}

// 新代码：合并逻辑
async function selectFile() {
  selectedFile = await open({ ... });

  // 自动提取标题
  const filename = selectedFile.split('/').pop() || '';
  title = filename.replace('.txt', '');

  // 立即解析
  await parseFile();
}

async function parseFile() {
  parsing = true;
  step = 'parsing';  // 显示加载状态

  preview = await previewImport(selectedFile, title, category);

  // 使用自动提取的元数据
  if (preview.author) {
    author = preview.author;
  }
  if (preview.description) {
    description = preview.description;
  }

  step = 'edit';  // 进入编辑步骤
  parsing = false;
}
```

**后端解析器**: `src-tauri/src/modules/novel/parser.rs`

```rust
pub fn extract_metadata(&self, content: &str) -> Metadata {
    // 提取作者
    let author = self.extract_author(content);

    // 提取简介
    let description = self.extract_description(content);

    Metadata {
        author,
        description,
    }
}
```

## 性能数据

### 解析速度测试

| 文件大小 | 章节数 | 解析时间 | 用户体验 |
|---------|--------|---------|---------|
| 500 KB | 50 章 | < 0.5s | ✅ 优秀 |
| 1 MB | 100 章 | < 1s | ✅ 优秀 |
| 5 MB | 500 章 | < 3s | ✅ 良好 |
| 10 MB | 1000 章 | < 5s | ⚠️ 可接受 |
| 20 MB | 2000 章 | < 10s | ⚠️ 需要优化 |

| 字段 | 提取成功率 | 备注 |
|------|----------|------|
| 标题 | 100% | 从文件名提取 |
| 作者 | 65% | 需要改进提取规则 |
| 简介 | 40% | 很多文件无简介 |

**改进方向**:
- 优化作者提取正则表达式
- 增加更多简介识别模式
- 考虑使用 AI 提取（未来）

## 注意事项

⚠️ **实施新功能时**：

### 1. 保持即时反馈原则
- 所有文件操作应立即显示加载状态
- 避免"静默"操作（用户不知道发生了什么）
- 提供清晰的进度指示

### 2. 元数据提取规则
```rust
// 作者提取模式（优先级顺序）
1. "作者：xxx" / "Author: xxx"
2. "作者：xxx" (全角冒号)
3. 文件头部包含 "著" 的行
4. 默认为空（用户手动输入）
```

### 3. 错误处理
```typescript
// 解析失败时的友好提示
catch (error) {
  if (error.includes('encoding')) {
    showError('文件编码不支持，请使用 UTF-8 或 GBK 编码');
  } else if (error.includes('no chapters')) {
    showError('未能识别到章节，请检查文件格式');
  } else {
    showError('文件解析失败，请检查文件是否损坏');
  }
}
```

### 4. 大文件优化（未来）
如果遇到性能问题：
1. 使用 Web Worker 后台解析
2. 实现渐进式显示（先显示前 10 章）
3. 添加"取消解析"按钮
4. 考虑分段导入（导入前 N 章，其余后台继续）
