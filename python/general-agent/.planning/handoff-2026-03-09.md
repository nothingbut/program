### TDD 流程
1. **先写测试** - 明确预期行为
2. **运行失败** - 验证测试有效
3. **最小实现** - 满足测试要求
4. **运行通过** - 验证实现正确
5. **重构优化** - 提升代码质量
6. **提交代码** - 频繁提交

### 双层审查
1. **规范审查** - 验证是否符合任务要求
2. **代码质量审查** - 验证代码质量（类型、测试、格式）
3. **修复循环** - 发现问题 → 修复 → 重审
4. **批准通过** - 所有问题修复后继续

### 异步实现
1. **一致性** - 所有 I/O 操作都异步
2. **aiosqlite** - 使用异步数据库驱动
3. **避免阻塞** - 不调用同步 I/O
4. **类型注解** - async 方法返回 Awaitable

### 测试策略
1. **正常路径** - 验证核心功能
2. **边界情况** - 空数据、不存在、未初始化
3. **错误处理** - 验证异常抛出
4. **Mock 使用** - 隔离外部依赖
