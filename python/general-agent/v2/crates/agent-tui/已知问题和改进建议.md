# Agent TUI 已知问题和改进建议

## 🐛 已知问题

### 1. UX 显示混乱

**问题描述**:
- 流式响应时，每个段落都被当作独立的消息添加
- 导致聊天窗口显示多条 assistant 消息，而不是一条完整消息
- 缺少滚动功能，长对话无法查看历史消息

**根本原因**:
```rust
// 当前实现：每个段落都创建新消息
BackendUpdate::ParagraphComplete { session_id, paragraph } => {
    self.state.add_message(session_id, MessageItem {
        role: "assistant".to_string(),
        content: paragraph,  // ❌ 每次都是新消息
        timestamp: chrono::Utc::now(),
    });
}
```

**改进方案**:

#### 方案1: 累积到一条消息（推荐）

修改状态管理，在流式响应时累积内容：

```rust
// 在 AppState 中添加
pub struct AppState {
    // ... 现有字段
    streaming_buffer: HashMap<Uuid, String>,  // 会话ID -> 累积的内容
}

// 处理流式响应
BackendUpdate::ParagraphComplete { session_id, paragraph } => {
    // 累积到缓冲区
    self.state.streaming_buffer
        .entry(session_id)
        .or_insert_with(String::new)
        .push_str(&paragraph);

    // 标记为流式状态
    self.state.set_session_state(session_id, SessionState::Streaming);
}

// 响应完成时才添加消息
BackendUpdate::ResponseComplete { session_id } => {
    if let Some(content) = self.state.streaming_buffer.remove(&session_id) {
        self.state.add_message(session_id, MessageItem {
            role: "assistant".to_string(),
            content,
            timestamp: chrono::Utc::now(),
        });
    }
    self.state.set_session_state(session_id, SessionState::Idle);
}
```

#### 方案2: 添加滚动支持

使用 Ratatui 的滚动功能：

```rust
// 在 AppState 中添加
pub struct AppState {
    // ... 现有字段
    scroll_offset: HashMap<Uuid, u16>,  // 每个会话的滚动位置
}

// 渲染时使用 Scrollbar
use ratatui::widgets::Scrollbar;

pub fn render_chat_window(f: &mut Frame, area: Rect, state: &AppState) {
    // ... 构建 lines

    let scroll_offset = state.scroll_offset
        .get(&session_id)
        .copied()
        .unwrap_or(0);

    let paragraph = Paragraph::new(lines)
        .block(block)
        .wrap(Wrap { trim: false })
        .scroll((scroll_offset, 0));  // 添加滚动

    f.render_widget(paragraph, area);
}
```

### 2. @greeting skill 不能工作

**问题描述**:
- 输入 `@greeting` 后没有执行技能
- conversation_flow 的技能处理可能未正确配置

**检查清单**:

1. **技能文件是否存在？**
   ```bash
   ls -la v2/crates/agent-skills/examples/test_skills/greeting.md
   ```

2. **conversation_flow 是否配置了技能路径？**
   ```rust
   // 检查 tui_demo.rs 中的配置
   let conversation_config = ConversationConfig::default();
   // ❌ 可能缺少技能路径配置
   ```

**修复方案**:

修改 `tui_demo.rs`，配置技能路径：

```rust
// 创建对话流程配置
let mut conversation_config = ConversationConfig::default();

// 设置技能目录路径
conversation_config.skills_directory = Some(
    std::path::PathBuf::from("crates/agent-skills/examples/test_skills")
);

let conversation_flow = Arc::new(ConversationFlow::new(
    session_manager.clone(),
    llm_client.clone(),
    conversation_config,
));
```

**验证**:
```bash
# 查看日志确认技能加载
RUST_LOG=info ./run_tui.sh

# 应该看到类似输出：
# INFO  Loaded skill: greeting
# INFO  Detected skill invocation: @greeting
```

## 🔧 优先修复建议

### 高优先级（影响基本使用）

1. **修复流式响应显示** - 方案1（累积消息）
   - 影响：每次响应显示10+条消息，难以阅读
   - 难度：中等
   - 时间：30分钟

2. **配置技能路径** - 修改 tui_demo.rs
   - 影响：技能功能完全不可用
   - 难度：低
   - 时间：10分钟

### 中优先级（改善体验）

3. **添加滚动功能**
   - 影响：长对话无法查看历史
   - 难度：中等
   - 时间：1小时

4. **添加消息时间戳显示**
   - 当前时间戳存在但不显示
   - 难度：低
   - 时间：15分钟

### 低优先级（锦上添花）

5. **自动滚动到底部**
   - 新消息时自动滚动
   - 难度：低
   - 时间：20分钟

6. **支持多行输入**
   - 当前只能单行
   - 难度：中等
   - 时间：1小时
